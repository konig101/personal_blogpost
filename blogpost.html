este es mi primer reporte de avance regirtrado en este blogpost

hoy aprendi a registrar mis avances en un git de la terminal y a usar css

si nesecito ayuda con un codigo desconocido en la terminal puedo mirar codigos para la terminal.pdf que se encuentra en la carpeta principal de git

agrande el codigo principal de css aprendiendo nuevas funciones que deberia repasar en el proximo curso

aprendi a crear ramas y trabajar mi proyecto por partes diferente haciendo que mi trabajo sea mas rapido y ayudandome a trabajar en equipo

hoy por fin pude cargar mi projecto a github y es mucho mas comodo registrar cambios y analizar mis avances desde aqui

recuerda que para entrar a github desde la terminal tienes que crear un token en ajustes y activar los permisos

para crear una llave ssh y asegurar mi proyecto en github con una mejor conexion interna tengo que generar el codigo ssh-keygen -t rsa -b 4096 -C "romangranada917@gmail.com"

despues de crear la keygen la puedo registrar con eval $(ssh-agent -s) y la puedo añadir con ssh-add ~/.ssh/id_rsa

recuerda si cambias de pc crea otra keygen y dale la llave publica github para entrelazar la comunicacion de manera segura

te recomiendo que dejes de estudiar a las 10pm si te queda algo pendiiente lo puedes hacer mañana es por tu salud

los codigos que has utilizado estos dias son muchos pero recurda que los puedes buscar con las flechas del tecado

recuerda para la configuracion del sistema remoto (por fuera de tu terminal) se usa git remote y si nesecitas algo mas puedes investigarlo

para crear ramas se usa git branch y para revisar las usas git checkout "nombre de la rama" sin las comillas

para agregar algo al staying o al origin se puede usar -a delante del commit y para gregarlo y registrarlo se usa git -am

recuerda para los comandos que son solo letras se usa -"y las letras necesarias" pero para los comandos que son palabras se usa -- y la palabra

revisa y actuliza siempre el github con los comandos de "git push" para enviar commits o informacion y "git pull" para traer

si en algun momento deseas crear un tag o una nota de version los puedes hacer con git tag -a "numero de la version" -m "mensaje de la version" y git push origin --tags para registrarlo en github

recuerde que si nesecitas ayuda en git tienes el hyperblog en la carpeta proyectodemo
